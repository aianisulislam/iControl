## What This Project Is

**iControl** is a macOS menu bar app that turns any LAN device with a browser into a wireless trackpad and keyboard for the Mac. Zero installation, no accounts, no cloud â€” browser connects to a local HTTP/WebSocket server and streams input commands.

**Core principles:**
- LAN-only (physically cannot leave the network)
- Zero client install (browser PWA)
- Input-only, one-way protocol (no screen data, no file access)
- No external dependencies anywhere (backend or frontend)
- Under 1MB total

**Current version:** 1.3
**Deployment target:** macOS 13.5+
**Build tool:** Xcode 15+ (no external package manager)

## WebSocket Message Protocol

**Client â†’ Server (auth â€” must be first message after WS open):**

| `type` | Required fields | Purpose |
|--------|----------------|---------|
| `auth` | `token` (string), `device` (string) | Authenticate with persistent or session token |
| `auth:request` | `device` (string) | Request approval via NSAlert when no valid token is available |

**Client â†’ Server (commands â€” only accepted after auth is approved):**

| `type` | Required fields | Purpose |
|--------|----------------|---------|
| `move` | `dx`, `dy` | Cursor delta movement |
| `dragMove` | `dx`, `dy` | Cursor move while left-button held |
| `scroll` | `dx`, `dy` | Two-axis scroll (pixel units) |
| `click` | `button` ("left"/"right"/"middle") | Single click |
| `mouseDown` | `button` | Button press |
| `mouseUp` | `button` | Button release |
| `key` | `key` (string, e.g. `"cmd+tab"`) | Legacy keyboard passthrough |
| `text` | `value` (string) | Text composition (sends chars) |
| `kb` | `key` (string), `flags` `{shift,ctrl,opt,cmd}` | New keyboard pipeline with simultaneous modifiers |
| `system` | `action` (string), optional `value` | System controls (see list below) |
| `app` | `app` (string: name or bundle ID) | Launch / focus app |
| `url` | `url` (string) | Open URL on Mac |

**System `action` values:**
`playPause`, `nextTrack`, `previousTrack`, `volumeUp`, `volumeDown`, `mute`, `setVolume` (+ `value` 0â€“100), `brightnessUp`, `brightnessDown`, `missionControl`, `appWindows`, `launchpad`, `minimize`, `fullscreen`, `delete`, `screenshot`, `siri`

**Server â†’ Client (auth responses):**

| `type` | Fields | Meaning |
|--------|--------|---------|
| `auth:approved` | `token` (string) | Connection accepted; client stores token in `sessionStorage` |
| `auth:rejected` | `reason`: `"invalid_token"` | Malformed or empty token â€” show manual entry modal |
| `auth:rejected` | `reason`: `"session_expired"` | Well-formed token not recognised â€” auto-send `auth:request` |
| `auth:rejected` | `reason`: `"denied"` | Host denied the request â€” lock tab, no further retries |
| `auth:open` | â€” | Server is in Open mode â€” no auth needed, hide modal |

**Server â†’ Client (state):**
```json
{ "type": "state", "volume": 75 }
```
Sent on connection open and whenever system volume changes.

## Key Component Details

### AuthContext (iControlApp.swift)
- Shared reference type passed from `iControlApp` â†’ `HTTPServer` â†’ each `WebSocketServer`
- `mode: String` â€” `"secure"` or `"open"`, mirrors `UserDefaults` key `"authMode"`
- `persistentToken: String` â€” 4-char token from `UserDefaults` key `"authToken"`, generated once on first launch
- `approvedSessions: Set<String>` â€” in-memory only, never persisted; cleared on every app launch and on token regeneration
- `isWellFormed(_:)` â€” true if token is exactly 4 uppercase alphanumeric chars
- `isTokenValid(_:)` â€” true if token matches `persistentToken` OR exists in `approvedSessions`
- `generateToken()` â€” picks 4 chars from unambiguous charset `ABCDEFGHJKLMNPQRSTUVWXYZ23456789`

### HTTPServer.swift
- `NWListener` on `0.0.0.0:4040`
- Accumulates request bytes until `\r\n\r\n` before parsing
- WebSocket upgrade: reads `Sec-WebSocket-Key`, SHA1-hashes it with magic UUID, base64-encodes, sends 101
- Serves resources from `Bundle.main` â€” path-traversal protected (blocks `..`)
- Tracks clients in `[ObjectIdentifier: NWConnection]` and `[ObjectIdentifier: WebSocketServer]`
- `broadcast(state:)` iterates all WebSocket instances to push volume state

### WebSocketServer.swift
- Manual RFC 6455 frame parsing â€” no library
- Variable payload lengths: 7-bit (â‰¤125), 16-bit (126), 64-bit (127)
- Client frames are always masked â€” XOR-unmask before JSON parse
- Holds a reference to `AuthContext`; per-connection `authState: ConnectionAuthState` (`.pending` / `.approved`)
- On `start()`: if mode is `"open"`, immediately sets state to `.approved` and sends `auth:open`; otherwise stays `.pending`
- While `.pending`: only `auth` and `auth:request` message types are processed; all others are silently dropped
- `auth` message: validates token via `AuthContext`; on success sets `.approved` and sends `auth:approved`; on well-formed-but-unknown sends `auth:rejected session_expired`; on malformed sends `auth:rejected invalid_token`
- `auth:request` message: dispatches `NSAlert` on main thread; Allow generates a new session token, inserts into `approvedSessions`, sets `.approved`, sends `auth:approved`; Deny sends `auth:rejected denied`
- Calls `InputController` directly for all approved command types

### InputController.swift
- **Mouse:** Tracks cursor position (`lastKnownPosition`) refreshed every 1s to avoid stale deltas
- **Drag:** Posts `leftMouseDragged` with `CGEventField` HID delta values
- **Scroll:** `scrollWheelEvent2Source` with `.scrollWheelEventDeltaAxis1/2`
- **Keyboard (legacy `pressKey`):** Parses `cmd+`, `shift+` prefix chains, maps to keyCode via `alphaNumericKeyCode()`, injects unmapped chars as UTF-16 unicode
- **Keyboard (new `handleKbInput`):** Direct keycode dispatch for F1â€“F12, arrows, specials, printable chars; applies `CGEventFlags` from `flags` dict; uses `.cgAnnotatedSessionEventTap` for Ctrl+Arrow to avoid conflicts
- **Modifier timing:** `usleep(1500)` between modifier key-down and the target key
- **Volume:** CoreAudio property listener on 3 channels; sets scalar on `kAudioDevicePropertyVolumeScalar`
- **App launch:** `NSWorkspace.shared.launchApplication` / `openURL` / Process for edge cases

### iControlApp.swift
- `MenuBarExtra` with QR code image + two URL labels
- QR code: `CIQRCodeGenerator` filter, 10Ă— scale, M error correction
- Hostname URL: `SCDynamicStoreCopyLocalHostName` â†’ `hostname.local:4040`
- IP URL: UDP socket to `8.8.8.8:80` + `getsockname()` â†’ `inet_ntop()`
- Launch-at-login: `SMAppService.mainApp` + `@AppStorage("launchAtLogin")`
- Auth mode persisted via `@AppStorage("authMode")` (`"secure"` / `"open"`)
- Auth token persisted via `@AppStorage("authToken")` â€” generated once on first launch
- Menu bar shows Connection Security submenu (chevron row, inline Secure/Open options with checkmarks) and Regenerate Token (hidden in Open mode)
- Switching to Open mode requires `LAContext.evaluatePolicy(.deviceOwnerAuthentication, ...)`; falls through silently if LA unavailable
- Switching to Secure mode auto-generates a new token and clears `approvedSessions`
- QR code URL and displayed URLs include `?token=XXXX` in Secure mode, bare URL in Open mode
- `authModePickerBinding` â€” custom `Binding<String>` used instead of `$authMode` + `.onChange` to avoid spurious LA auth triggers when MenuBarExtra window re-opens

### index.html (Frontend)
- Single self-contained file â€” no bundler, no framework, no imports
- Dark theme (`--color-bg: #0b1220`), glassmorphism, touch-optimized buttons (36â€“48px)
- Three main tabs: **General** (modifier keys + key buttons), **System** (media/system controls), **Apps** (launcher + keyboard)
- Full virtual QWERTY keyboard layout (hidden panel, toggled in)
- `touchTracker` â€” unified gesture recognizer: tap, hold, swipe, drag, scroll, two-finger scroll, orientation-aware coordinate transform
- Sticky modifier keys (Cmd, Shift, Option, Ctrl) with visual toggle state
- Volume/brightness sliders with pointer-capture dragging
- Settings drawer: sensitivity slider
- **Auth flow:** on WS open, immediately sends `auth` message with token from `sessionStorage("icontrol-session")` â†’ `localStorage("icontrol-token")` â†’ `""`; URL `?token=` param is stripped on load and stored to localStorage
- **Auth modal:** shown when token is rejected; contains token input + Connect button + Request Access button; in denied state, all buttons are hidden and a denial message is shown
- **Denied lock:** `sessionStorage("icontrol-denied")` flag; once set, all reconnect paths (`connectWebSocket`, `checkAndReconnectWebSocket`, visibility change) bail immediately for the tab's lifetime

### sw.js (Service Worker)
- Cache name: `icontrol-cache`
- Pre-caches manifest, favicons, icons on install
- Network-first with cache fallback on fetch
- Detects version changes via `<meta name="version">` tag, notifies client via `postMessage`

## Conventions & Patterns

**Swift:**
- Weak self in all closures that capture `self`
- `DispatchQueue.main` for UI; background queues for network
- `@AppStorage` for persisted preferences
- `ObjectIdentifier` as dictionary key for connections
- Errors are handled silently (guard/return pattern) â€” no throws propagated to callers

**JavaScript (index.html):**
- Vanilla ES2020+, no libraries
- State stored in module-level `let`/`const` variables
- localStorage for UI preferences (tab selection, sensitivity, etc.)
- All WebSocket sends go through a single `send(obj)` helper that JSON-stringifies
- Touch events normalized through `touchTracker` before any action

**Protocol:**
- All messages are JSON text frames
- Client never receives input commands â€” server only pushes state and auth responses
- First client message after WS open must always be `auth`; all command messages are silently dropped until auth is approved

## Security Model

- Server binds `0.0.0.0:4040` â€” reachable on all LAN interfaces, not exposed to internet
- Two auth modes: `"secure"` (default, token required) and `"open"` (no auth)
- Switching to Open requires biometric/password via `LocalAuthentication`
- Persistent token: 4-char, stored in `UserDefaults`, embedded in QR code URL as `?token=`
- Session tokens: generated per `auth:request` approval, in-memory only (`approvedSessions`), cleared on app relaunch or token regeneration
- Denied clients: `sessionStorage` flag prevents any reconnect for the tab's lifetime
- Protocol is input-only â€” no file access, no screen capture, no system info returned beyond volume
- CGEvent requires Accessibility permission â€” user must explicitly grant
- Path traversal in HTTP server is blocked (`..` sequences rejected)

## What NOT to Change Without Care

| Area | Risk |
|------|------|
| `InputController.alphaNumericKeyCode()` | 100+ key mappings; easy to break keystrokes for specific symbols |
| WebSocket frame parser in `WebSocketServer.swift` | Manual RFC 6455 â€” masking, length encoding are subtle |
| `inject_version.sh` | Must match the `<meta name="version">` tag name in `index.html` |
| Service Worker cache key in `sw.js` | Changing cache name forces all clients to re-download |
| `touchTracker` gesture logic in `index.html` | Complex state machine; orientation transforms make it non-obvious |
| `usleep(1500)` in keyboard modifier sequencing | Removing it breaks modifier+key combos on some apps |
| `authModePickerBinding` in `iControlApp.swift` | Uses custom `Binding` instead of `$authMode + .onChange` â€” deliberately avoids spurious LA auth on MenuBarExtra re-open; do not replace with `.onChange` |
| `icontrol-denied` in `sessionStorage` | Checked at every reconnect path â€” removing any one check re-opens the denied lockout loophole |
| Auth gate in `WebSocketServer.handleText` | The `authState == .pending` guard must remain the first check â€” moving it below any command dispatch would allow unauthenticated input |
